# Landing Page Content Refresh Design

## Goal

Refresh the landing page so it matches the product that now exists in production:

- `MYP` as a bridge-tier reading and inquiry workflow
- `DP` as an academic framing and argument-building workflow

The page should still feel like the same brand, but the copy should stop speaking like an older `PYP/MYP` or generic IB-reading offer.

## Scope

This slice updates four homepage areas only:

- `Hero`
- `Solution`
- `How it works`
- `FAQ`

The current layout, pricing section, integrations section, and footer structure should remain intact.

## Product Problems To Fix

1. The homepage says `MYP + DP`, but most of the copy still sounds generic.
2. The FAQ still contains outdated `P5 to MYP` language.
3. The `TED-Talk Structure` card no longer reflects the live product model.
4. The setup section leaks implementation details like `09:00 UTC` instead of explaining the family workflow.

## Content Direction

### Hero

The hero should clearly communicate the two-tier academic promise:

- `MYP`: inquiry, global context, compare-or-connect thinking
- `DP`: abstract, claim, evidence, TOK-style thinking

It should sound calm and credible, not overly promotional.

### Solution

The three cards should describe real product outcomes:

1. `MYP bridge reading`
2. `DP academic framing`
3. `Calm delivery, lasting record`

This keeps one shared brand skeleton while making the programme split obvious.

### How It Works

The setup flow should read from the family perspective:

1. choose learning stage and interests
2. connect the preferred reading/archive tools
3. receive, reflect, and save inside the ongoing learning loop

Avoid brittle implementation details such as exact UTC timestamps.

### FAQ

The FAQ should become the clearest place where the homepage answers:

- who the product is for now
- how `MYP` and `DP` differ
- how Goodnotes works
- how Notion works
- whether both tools are required
- how subscription cancellation works

## Non-goals

This slice does not include:

- changing the page layout
- changing pricing structure
- changing integrations ordering
- changing footer social proof
- adding new screenshots or assets

## Success Criteria

- The homepage speaks consistently as a `MYP + DP` product
- Old `PYP/P5 to MYP` positioning is removed from the FAQ
- The solution section reflects real runtime product behavior
- The setup flow explains the user journey instead of internal scheduling details
- Regression tests catch future copy drift back to outdated positioning
